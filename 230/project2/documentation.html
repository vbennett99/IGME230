Our Process

  1. We chose the recipe API, created an idea for a search engine for Breath of the Wild recipes.
  2. This site would present the user with BotW recipes along with real life recipes that were similar.
  3. We then created a simple site for our prototype which showed that the API was accessible.
  4. We input each recipe with its name, type, and ingredients.
  5. Then searched the API for recipes that were similar to the fictional ones.
  6. We added more search options to narrow the user's search.
  7. We polished the site more with a banner, CSS, and a custom BotW font.

Requirements We Met

  • HTML and CSS are validated.
  • Control requirements are met for filtering results.
  • Adjusted CSS to make the browser easy to navigate and look at.
  • Our design is responsive, looks good on a range of displays.
  • Widgets are accurately labeled.
  • No errors are thrown.

Known Errors

  • The API should in theory work and will return data when stringified, but when actually attempting to retrieve data for use, does not compute.
  • Last saved term is not stored in local storage.
